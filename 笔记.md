# 项目概览：

scGPT = 一个“单细胞数据的 Transformer 模型” + 一套任务 pipeline  

输入：单细胞表达矩阵。过程：预处理 → token 化 → Transformer 微调；输出：细胞的表示 / 类型 / 扰动预测  

整个项目分为两层：  

模型层（core model）  
- Transformer / embedding  
- tokenization（gene → token）  
- forward / loss  

任务层（tutorials）  
- annotation（分类）  
- integration（批次校正）  
- perturbation（扰动预测）  

文件夹结构：  

1️⃣ data层 → 数据（刚下载的 h5ad）filtered_ms_adata.h5ad是单细胞表达矩阵，c_data.h5ad是control  

2️⃣ tutorials层 → 使用方法（现在要跑的）Tutorial_Annotation.ipynb 将输入的表达矩阵进行分类AnnData → token化 → Transformer → 分类 → 评估  

3️⃣ scgpt核心层 → 模型 + tokenizer + training逻辑  

4️⃣ docs/tests → 文档 + 测试（可以暂时忽略）

# 3. 参数设置

1. n_bins=51,为什么要 binning  

原始表达值是连续数值，但 scGPT 这类模型常会把表达量映射到离散区间，以便更适合 token/embedding 的处理方式。51是50 个表达区间再加一个特殊值/边界。表达值不会直接原样喂进模型，而是先做分箱。  

2. 为什么要做dropout？  

0.2表示训练时有 20% 的神经元输出会被随机丢弃。为了减少过拟合。  

3. schedule_ratio=0.9,是在干嘛？  

在训练深度学习模型时，我们通常不使用恒定的学习率，而是采用 Warmup（预热）+ Decay（衰减） 的策略。  

• Warmup（预热）阶段：学习率从 0 慢慢增加到设定的最高值，防止模型在刚开始训练时因为梯度太大而崩掉。  
• Decay（衰减）阶段：学习率从最高值慢慢降到 0，让模型在训练后期能够更精细地寻找最优解。  

schedule_ratio则定义了Warmup 阶段占据总训练步数（Total Steps）的比例  

4. pre-norm是什么？  

是 层归一化（Layer Normalization） 放置位置的一种策略。Transformer 每一层通常由“注意力机制（Attention）”和“前馈网络（FFN）”组成，而 Pre-norm 决定了什么时候对数据进行“标准化”处理。Pre-norm vs. Post-norm  

在 Transformer 的演进史上，主要有两种放置方式：  

• Post-norm (原始 Transformer 采用)：数据先经过 Attention，然后进行残差连接（Add），最后再做归一化（Norm）。  
    ◦ 公式：$Layer(x) = Norm(x + Sublayer(x))$  
• Pre-norm (scGPT 及大多数现代大模型采用)：数据在进入 Attention 或 FFN 之前，先进行归一化。  
    ◦ 公式：$Layer(x) = x + Sublayer(Norm(x))$  

① 极大的训练稳定性（无需漫长的 Warmup）  

Post-norm 在层数加深时，输出端的梯度容易变得非常大，导致模型在训练初期极不稳定。这也是为什么早期模型需要极其精细的 Learning Rate 设置。 Pre-norm 相当于给每一层都装了“减震器”，梯度在传递过程中更加平滑。这对于参数量巨大的 scGPT 来说，能有效防止你之前担心的训练崩溃或 NaN 报错。  

② 允许更深的网络结构  

研究发现，使用 Pre-norm 的 Transformer 即使不使用复杂的学习率调度策略，也能轻松扩展到 100 层以上。在处理复杂的生物学特征（如数万个基因的相互作用）时，这种深度非常重要。

# 4. 下载模型scGPT_human model(whole_human)  

```text
scGPT-main/
├── save/
│   └── scGPT_human/
│       ├── best_model.pt
│       ├── vocab.json
│       ├── args.json
```
args.json: 文件类型：JSON (JavaScript Object Notation),它记录了模型训练时的所有“超参数”（层数、维度等），当你加载模型时，程序会先读这个文件，按照当初定义的规格“搭好架子”，然后才能把权重填进去。  

best_model.pt: PyTorch 指向文件，通常是二进制格式,存储了模型在训练过程中学到的所有权重参数（Weights）  

vocab.json: JSON 格式的文本文件,基因名称对应数字 ID 的索引表，这个文件定义了：第 1 号基因是谁，第 500 号基因是谁。  

.h5ad: 是一种专门为单细胞数据设计的 HDF5 格式（Hierarchical Data Format）。它不仅仅包含基因表达量矩阵，还包含细胞的元数据（比如细胞来源、批次）、基因的注释信息，以及降维后的坐标（如 UMAP）。  

• 核心组件（AnnData 对象）：  
◦ X: 表达量矩阵（细胞 × 基因）。  
◦ obs: 细胞属性表（比如：这个细胞属于哪个大脑区域）。  
◦ var: 基因属性表（比如：基因全名）。  
◦ uns: 非结构化数据（比如：训练时的颜色设置、PCA 参数）。

### scGPT 表达量输入模式 (input_style) 对比

| 模式 | 形式 | 稳定性 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **normed_raw** | 原始归一化 | 差 | 极少数需要精确物理量分析的场景 |
| **log1p** | 对数变换 | 中 | 传统统计分析与简单神经网络 |
| **binned** | **离散分箱** | **优** | **预训练大模型 (scGPT) 的默认首选** |

4.scIB是什么？do_eval_scib_metrics = True  

scIB (single-cell Integration Benchmark) 是一套专门用于评价单细胞数据整合（Integration）效果的标准化指标体系。true意味着模型性能的评估不再局限于细胞分类准确率，而是通过计算 ASW、NMI、ARI 以及 LISI 等定量指标，综合衡量模型在低维嵌入空间中去除实验批次效应（Batch Removal）与保留生物学异质性（Biological Conservation）之间的平衡能力，从而全面评价数据整合后的表征质量及其对下游生物学分析的可靠性。
